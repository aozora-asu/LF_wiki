# 日勤: 災害時初動マニュアル

大規模災害や広域障害が発生した場合の日勤チームの行動フローをまとめています。外部インフラの復旧待ち時間でも、情報整理と社内連絡を止めないことが重要です。

## 0.  安否と連絡体制の確認

1. `#bcp` チャンネルで安否報告スレッドを立ち上げ、日勤メンバーの状況を回収。  
2. 通話が必要な場合は「災害用音声ブリッジ」を利用し、会議 ID をスレッドに固定。

## 1. 情報の一次集約

- 内閣府・気象庁など公的発表の一次ソースリンクを 5 分ごとに更新。  
- 自社サービスの障害状況は監視ツールのスクショとログ概況をまとめて `#status` に投稿。  
- 取材班には速報優先・安全最優先で外出可否を判断して伝達する。

## 2. 役割分担

| 役割 | 担当タスク |
| ---- | ---------- |
| 連絡係 | 役員・夜勤・広報への定時連絡（30 分間隔） |
| 原稿係 | 速報記事テンプレの更新、社内向け Q&A の整備 |
| ロジ係 | 交通機関・出社可否の把握、在宅勤務に切り替える場合の手続き |

## 3. 夜勤への引き渡し

- 災害ログを `manuals/entries/day/disaster.md` の末尾にコピーし、時間軸を残す。  
- 優先度 A/B/C を明記したトランジションメモを `#shift-handoff` に投稿。  
- 必要なら臨時ミーティングを設定し、夜勤リーダーに直接ブリーフィングする。

## 4. 見直し

- 収束後 24 時間以内に今回の対応を振り返り、改善案を Wiki に追記。  
- 新しいチェックリストやテンプレが必要な場合は `manuals/new_page.sh`（後述）で素早くページを増やしてください。
